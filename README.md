## UI Theme & Concept

**การตัดสินใจ:**  
ธีมที่เลือกสำหรับเกมคือ **MMORPG แฟนตาซียุคกลาง** ซึ่งสอดคล้องกับชื่อเกม *Quest & Chest* และร้านค้า *Dungeon Mart* การวิเคราะห์ชื่อเกมและร้านค้าช่วยให้กำหนดทิศทางของโปรเจคและเลือก Asset จาก Unity Asset Store ได้ตรงตาม Concept  

**เหตุผล:**  
ธีมนี้เข้ากับตัวเกม และ Asset ที่เลือกใช้มีไอเท็มและปุ่มหลากหลาย เหมาะสมกับธีม ทำให้ UI ของร้านค้ากลมกลืนกับ Concept ของเกม  

**ข้อสรุปข้อคิด:**  
การเลือก Asset และการออกแบบ Concept ควรทำให้ **Art ของ UI อยู่ในทิศทางเดียวกัน** สร้างความรู้สึกที่ดีให้ผู้เล่น และช่วยเพิ่มประสบการณ์ใช้งาน (UX) ให้สอดคล้องกับธีมเกม

## Tools & Plugins

ในการพัฒนา UI และระบบของเกม *Quest & Chest* ผมเลือกใช้เครื่องมือและ Plugins ดังนี้:  

---

### 1. DoTween
- **การตัดสินใจ:**  
  ผมเลือกใช้ DoTween เพราะว่ามันสามารถทำ Animation ของ UI และ Object ต่าง ๆ ได้ **สมูทและยืดหยุ่น** กว่าการใช้ Animator แบบปกติ  

- **เหตุผล:**  
  - ลดความซับซ้อนของการสร้าง Animation Clip และการจัด Keyframe และ Animatate ของ Dotween มันดูลื่นไหลแบบธรรมชาติ  
  - ใช้โค้ดควบคุม Animation ได้ง่ายและชัดเจน  
  - เหมาะสำหรับการทำ Transition ของ UI เช่น ป๊อปอัพ, Fade, Move, Scale  

- **ข้อสรุปข้อคิด:**  
  การใช้ DoTween ทำให้ **Workflow ของ UI Animation เร็วขึ้นและเป็นระบบ** ช่วยให้ UI ตอบสนองได้ดีขึ้นและเพิ่มประสบการณ์ใช้งาน (UX)  

- **ปัญหาที่พบเจอ:**  
  ต้องใช้เวลาในการเรียนรู้การใช้งาน Plugin พอสมควร และพบปัญหาเมื่อมีการเรียกใช้งาน Animation ซ้ำ ๆ ติดต่อกัน ทำให้เกิดอาการกระตุกหรือเฟรมเรตตก  
  อีกทั้งยังเจออาการ **Tween ค้าง** เวลาหยุดการทำงานใน Unity Editor แล้วกลับมารันใหม่ ซึ่งเกิดจาก Tween เก่าที่ยังไม่ถูก Clear หรือ Kill ส่งผลให้บางครั้งระบบ Animation ทำงานผิดพลาด ซึ่งผมจำเป็นต้องศึกษาและจัดการการเคลียร์ Tween ให้ถูกวิธีมากขึ้น

---

### 2. UniTask
- **การตัดสินใจ:**  
  เลือกใช้ UniTask เพื่อจัดการ **Async Operation** ภายใน Unity เช่น การโหลด Asset, การรอ Event หรือการทำงานเบื้องหลัง ซึ่งเป็นงานที่ต้องรอเวลาและทำหลายอย่างพร้อมกัน  

- **เหตุผล:**  
  ผมมองว่าเป็นสิ่งที่ควรเรียนรู้ เพราะสามารถนำไปประยุกต์ใช้งานจริงในอุตสาหกรรมเกมได้  

- **ข้อสรุปข้อคิด:**  
  การใช้ UniTask ช่วยให้ **การจัดการงานที่ต้องรอเวลาเป็นระบบมากขึ้น** โค้ดสั้น กระชับ และไม่ซับซ้อนเท่า Coroutine เหมาะสำหรับเกมที่มีหลายระบบทำงานคู่ขนานกัน  

- **ปัญหาที่พบเจอ:**  
  ต้องใช้เวลาในการทำความเข้าใจการเขียนแบบ async/await ใน Unity เนื่องจากเพิ่งเริ่มใช้ จึงยังต้องศึกษาพอสมควร อีกทั้งยังพบปัญหา **Memory Leak** หรือ Task ที่ยังคงทำงานอยู่แม้กดหยุด Run ไปแล้ว ทำให้จำเป็นต้องออกแบบโครงสร้างโค้ดให้รอบคอบและจัดการ Task ให้ถูกวิธี

## Layout

### 1. Menu
<img width="565" height="1001" alt="Menu UI" src="https://github.com/user-attachments/assets/f5a79ccd-2f34-4016-838c-1f5e4d917978" />  

- **การตัดสินใจ:**  
  - วาง Layout แบบเรียบง่าย โดยเน้นปุ่มหลัก (*Play, Inventory, Shop*) ไว้ตรงกลาง เพื่อให้ผู้เล่นเข้าถึงได้ทันที และเหมาะสำหรับการทำ **Tutorial แนะนำการเล่น**  
  - จัดวางปุ่มเสริม เช่น **Effect Audio, Music Audio, Clear Data** ไว้ด้านล่าง เพื่อไม่ให้รบกวน UI หลัก  
  - แสดง **Money** ที่มุมซ้ายบน เพื่อให้ผู้เล่นเห็นค่าเงินได้ตลอดเวลา  

- **เหตุผล:**  
  หน้า Menu เป็น **First Impression** ของผู้เล่น การออกแบบให้ชัดเจนและเป็นระเบียบช่วยให้ผู้เล่นเข้าใจระบบได้ทันที และทำให้การสร้าง Tutorial ง่ายขึ้น

### 3. Shop
<img width="564" height="1002" alt="Shop UI" src="https://github.com/user-attachments/assets/c8422cc3-2c98-4147-b060-854e9cb9c269" />  

- **การตัดสินใจ:**  
  วางสินค้าเป็น **Grid** และใช้ **Item Card** ประกอบด้วย:  
  - รูปภาพ Item  
  - ชื่อ Item  
  - รายละเอียด Item  
  - ราคา Item  
  - ปุ่มซื้อ Item  

- **เหตุผล:**  
  การรวมปุ่มซื้อใน Item Card ทำให้ผู้เล่นสามารถเลื่อนดูสินค้าและกดซื้อได้สะดวก  
  นอกจากนี้ยังช่วยให้ระบบ Save/Load Inventory และล็อคปุ่มซื้อหลังซื้อสำเร็จ ทำงานง่ายขึ้น

### Problems (ปัญหาที่พบเจอ)
- การวาง Layout ต้องคำนึงถึง **Responsive** เพื่อให้รองรับหลายขนาดหน้าจอโดยไม่ส่งผลต่อ UI ที่ออกแบบไว้ ซึ่งผมก็ใช้เวลาแก้พอสมควรกว่าจะปรับให้เข้ากันได้
